새로운 아이디어는 의외로 직접 겪은 작은 불편함과 꼬리를 무는 호기심에서 발굴된다고 생각합니다.

제가 불편했던 경험은 다른 사람도 비슷하게 느낄 수 있다고 생각합니다. 그래서 반복되는 작업이나 기술적으로 궁금한 점을 마주했을 때, 어떻게 해결할 수 있을지 자주 고민하곤 했습니다. 그런 고민들을 직접 프로젝트로 구현해보며 하나씩 경험을 쌓았고, 제가 만든 작업물이 누군가에게는 작은 도움이 되길 바라는 마음이 생겼습니다. 완성도가 아쉬운 예전 프로젝트도 제가 어떤 문제를 고민했고 어떻게 풀어가려 했는지 보여주는 과정이라고 생각해, 저의 경험과 해결 과정을 GitHub에 남기고 싶었습니다.

무언가를 만들다 보면 생각처럼 굴러가지 않거나, 마음대로 풀리지 않을 때가 있습니다. 게임 캐릭터는 경험치가 숫자로 보이지만, 현실은 그렇게 단순하지 않았습니다. 내가 제대로 성장하고 있는지, 같은 자리에서 허우적거리고 있는 것은 아닌지, 이 고통을 얼마나 더 견뎌야 앞으로 나아갈 수 있는지 알려주는 기준조차 없었습니다. 그래서 저는 직접 시도하고, 막히고, 한계에 부딪혔을 때 비로소 경험치가 쌓인다고 말하고 싶습니다.

저는 시행착오 속에서 제 방향을 찾고, 실패 앞에서도 쉽게 꺾이지 않으며, 끝까지 붙잡고 한 단계씩 나아가는 개발자로 성장해가고 싶습니다.
