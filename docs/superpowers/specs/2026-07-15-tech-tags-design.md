# 기술 태그 SVG 설계 (2026-07-15)

README 프로필에 노션 태그 스타일의 기술 태그를 추가하는 설계다. 기존에 주석 처리된 skillicons.dev 아이콘 나열을 대체한다.

## 목적

언어 통계(metrics)만으로는 방문자가 "이 사람이 어필하려는 기술"을 읽을 수 없다. 크게 나열된 아이콘 대신, 노션 태그처럼 작고 간결한 필(pill) 형태로 핵심 기술만 보여준다.

## 결정 사항 요약

| 항목 | 결정 | 근거 |
|------|------|------|
| 태그 내용 | 기술명만 8개 (분야/직무 텍스트 없음) | 사용자 확정 |
| 필 스타일 | 모노톤 (org 카드 팔레트 재사용) | 테마 중립 원칙, 기존 카드와 톤 통일 |
| 아이콘 | 필 왼쪽에 14x14 기술 아이콘, 모노톤 재색칠. **선택 사항** (없으면 텍스트만) | 노션 태그의 색 점 자리를 아이콘이 대체 |
| 구현 | 태그별 SVG 파일 + 생성 스크립트 (CI 없음). 스크립트가 README 마커 구간까지 동기화 | org 카드와 동일 패턴, 자연 줄바꿈, 배열=단일 소스 |
| 배치 | 소개 문단 아래, metrics 2단 레이아웃 위 | 통계보다 먼저 지향점을 보여줌 |

## 태그 목록과 아이콘 소스

아이콘은 devicon(MIT 라이선스)의 `-plain` 변형을 기본 소스로 쓴다. plain 변형은 단색 치환용으로 설계돼 있다.

| 표시 이름 | slug | 소스 (devicon) |
|-----------|------|----------------|
| C# | csharp | `icons/csharp/csharp-plain.svg` |
| C/C++ | cplusplus | `icons/cplusplus/cplusplus-plain.svg` |
| .NET | dotnet | `icons/dot-net/dot-net-plain.svg` |
| Unity | unity | `icons/unity/unity-plain.svg` |
| Python | python | `icons/python/python-plain.svg` |
| TypeScript | typescript | `icons/typescript/typescript-plain.svg` |
| PostgreSQL | postgresql | `icons/postgresql/postgresql-plain.svg` |
| Docker | docker | `icons/docker/docker-plain.svg` |

특정 slug에 plain 변형이 없으면(구현 시 확인) simpleicons 같은 **다른 단색 소스**의 URL로 대체한다. devicon original 같은 다색 변형은 대체 소스로 쓰지 않는다. 레이어를 겹쳐 문양을 만드는 다색 아이콘은 단색 치환 시 레이어가 한 덩어리로 합쳐져 내부 문양이 사라지기 때문이다 (아래 paint 정규화의 다색 검출이 이런 소스를 거부한다).

## 비주얼 스펙

```
( [아이콘 14x14]  C#  )      <- 아이콘 있는 태그
(  C#  )                     <- 아이콘 없는 태그 (아이콘은 선택 사항)
```

- 필: 높이 24px 스타디움(`rx=12`), 좌 패딩 10, 아이콘-글자 간격 6, 우 패딩 12
- 아이콘이 없는 태그는 아이콘 영역 없이 좌우 패딩 12로 텍스트만 그린다
- 필 배경/테두리: `#8b949e` 채움 7% + 테두리 40% 1px (org 카드 `.card`와 동일 값)
- 글자: 400 12px, `#7d8590`, 폰트 스택은 org 카드와 동일 (`-apple-system,...,sans-serif`)
- 아이콘: paint 정규화 규칙으로 `#7d8590` 모노톤화 (아래 "아이콘 재색칠" 절 참고)
- 테마 중립 단일 팔레트, 미디어 쿼리 없음 (Safari `<img>` SVG 미디어 쿼리 미지원 함정 회피, PROJECT.md 참고)
- 캔버스: 폭 = 필 폭 + 우측 투명 여백 2px, 높이 = 24 + 하단 투명 여백 6px = 30px
  - 태그 사이 가로 간격 = 내장 여백 2px + README 어절 공백 약 4px = 약 6px
  - 하단 여백은 좁은 화면에서 줄바꿈될 때 세로 간격을 만든다

브랜드 컬러 아이콘 안은 기각했다. Unity 로고가 검정이라 다크 페이지에서 사라지고, 모노톤 필 원칙과도 어긋난다.

## 생성기 설계 (`scripts/gen-tech-tags.sh`)

수동 실행 스크립트다. CI를 두지 않는 이유: org 카드와 달리 동적 데이터(리포 수 등)가 없어 재생성 트리거가 "태그 목록 수정"뿐이고, 자동 커밋 소스를 늘리지 않기 위해서다.

### 입력 형식

스크립트 상단 배열에 한 항목당 1~3필드로 선언한다. 허용 형태는 세 가지뿐이다.

```bash
tags=(
  "C#|csharp|https://raw.githubusercontent.com/devicons/devicon/master/icons/csharp/csharp-plain.svg"
  "Rust"           # 1필드: 아이콘 없는 텍스트 필
  "MyLib|mylib"    # 2필드: icons/mylib.svg가 이미 있어야 함 (직접 넣은 아이콘)
)
```

- **URL이 있으면 slug는 필수다.** `이름||URL` 같은 slug 없는 URL 입력은 캐시 저장 경로(`icons/<slug>.svg`)를 정할 수 없으므로 오류로 거부한다.
- **slug는 `^[a-z0-9][a-z0-9-]*$`만 허용한다.** slug가 그대로 `icons/<slug>.svg`, `tag-<slug>.svg`, README `src` 경로가 되므로, `../` 같은 경로 조작이나 공백/따옴표/대문자가 섞이면 다른 파일을 덮어쓰거나 잘못된 HTML을 만든다. 정규식 밖이면 오류로 멈춘다.
- **표시 이름은 문자폭 표가 정의된 ASCII 집합**(영대소문자, 숫자, `#`, `+`, `.`, `/`, `-`, 공백)만 허용한다. 밖의 문자가 있으면 오류로 멈춘다. 폭 계산과 파일명 파생이 보장되는 범위로 입력을 좁히는 계약이다.
- 파일명은 slug가 있으면 `tag-<slug>.svg`, 없으면 표시 이름을 소문자화하고 영숫자 외 문자를 `-`로 치환해 만든다. 정규화 결과가 비거나, **최종 파일 경로(캐시/산출물)가 다른 항목과 충돌하면** 오류다.

### 아이콘 캐시 (`icons/` 폴더)

- `icons/<slug>.svg`가 없고 URL이 있으면 **임시 파일로 내려받아 검증한 뒤 원자적으로 이동**한다. 검증: HTTP 실패(`curl -f`)면 중단하고, **문서 루트가 `<svg>` 요소인지**(XML 선언/주석/DOCTYPE을 제외한 첫 요소 기준) 확인하며, **viewBox 4값이 숫자로 파싱되고 w/h가 양수인지**까지 본다. 단순 "`<svg` 문자열 포함" 검사는 SVG 로고가 박힌 HTML 페이지도 통과하므로 문서 단위로 판정한다. 실패 시 캐시에 아무것도 남기지 않는다.
- **외부 리소스 참조가 있으면 거부한다**: `href`/`xlink:href`의 http(s) URL, `url(http...)` 형태의 filter/paint 참조 등. `<img>` 임베드에서는 외부 리소스가 로드되지 않아 아이콘이 비거나 일부만 그려진다. 문서 내부 `#id` 참조와 `data:` URI는 허용한다.
- 기존 캐시 파일도 매 실행 같은 검증을 통과해야 한다. 손상돼 있으면 URL이 있을 때는 재다운로드하고, 없을 때는 명확한 오류로 멈춘다.
- 검증을 통과한 원본 파일은 커밋한다. 이후 재생성은 네트워크 없이 캐시로만 동작한다 (소스 핀 고정, 외부 서비스 장애와 무관).
- **제3자 라이선스 고지를 함께 커밋한다**: devicon 원본과 그 파생물(태그 SVG)을 공개 저장소에 배포하므로 MIT 고지 보존 의무가 있다. `icons/LICENSE-devicon.md`에 저작권/허가 고지 전문과 출처 링크를 두고, 다른 소스를 쓰게 되면 소스별 고지 파일을 추가한다.
- 직접 아이콘 파일을 `icons/`에 넣어도 된다 (유효한 파일이 있으면 URL은 무시). 이때도 라이선스 고지를 같이 둔다.

### 임베드: base64 래스터가 아니라 벡터 인라인

아이콘 SVG에서 viewBox와 내부 요소를 추출해 태그 SVG 안에 `<g transform="translate(...) scale(...)">`으로 직접 넣는다. org 카드가 GitHub 로고를 패스로 내장한 것과 같은 패턴이다.

- `<img>` 임베드 SVG는 외부 리소스를 로드하지 못하므로 인라인이 필수다.
- 벡터라서 래스터 변환 없이 14x14로 깨끗하게 스케일된다.
- viewBox가 소스마다 다르므로(devicon 128, simpleicons 24) 스크립트가 **viewBox 4값(min-x, min-y, w, h)을 전부 파싱**해 변환을 계산한다. min-x/min-y를 translate로 보정하고(원점이 0 0이 아닌 소스 대응), `14 / max(w, h)` 균일 스케일로 왜곡 없이 축소하며, 비정사각형 소스는 짧은 축을 14x14 영역 안에 중앙 정렬한다.

### 아이콘 재색칠 (paint 정규화)

단순 일괄 문자열 치환이 아니라 다음 규칙으로 정규화한다. `fill="none"`까지 색으로 덮으면 투명이어야 할 영역이 채워져 아이콘이 깨지기 때문이다.

- `fill="none"` / `stroke="none"`은 보존한다.
- 그 외 fill/stroke 속성값(색상 코드, `currentColor` 포함)은 `#7d8590`으로 치환한다.
- **다색 검출**: `none`을 제외한 fill/stroke 색이 2가지 이상이면 다색 아이콘으로 보고 오류로 멈춘다. 다색 레이어를 한 색으로 칠하면 내부 문양이 사라진 실루엣만 남는다.
- **루트 `<svg>`의 표현 속성도 전달한다**: 루트에 `fill`/`stroke`가 선언된 아이콘(예: `<svg fill="none" stroke="currentColor">`)은 내부 요소만 추출하면 그 속성이 사라져 윤곽선 아이콘이 채워진 도형으로 변한다. 루트의 fill/stroke를 같은 정규화 규칙으로 래퍼 `<g>`에 옮긴다.
- 인라인 내용을 감싸는 `<g>`에 `fill="#7d8590"`을 줘서, fill 속성이 아예 없는 요소도 상속으로 칠해지게 한다 (루트 전달 속성이 있으면 그쪽이 우선).
- `<style>` 블록, `class` 기반 색 지정, **인라인 `style` 속성**을 쓰는 아이콘은 지원 범위 밖이다. 스크립트가 감지하면 오류로 멈춘다 (조용히 깨진 아이콘을 만들지 않는다). 특히 인라인 `style="fill:..."`은 래퍼 `<g>`의 상속 fill보다 우선해 모노톤 보장을 조용히 깨므로 반드시 거부한다. devicon plain 계열은 속성 기반 단색이라 이 계약 안에 들어온다.

### 텍스트 폭 계산

SVG는 텍스트 자동 크기 조정이 없으므로 필 폭을 스크립트가 계산한다. 12px 기준 문자폭 테이블(0.1px 단위 정수 연산)로 근사한다. 대상 문자는 입력 계약이 허용하는 ASCII 집합(영대소문자, 숫자, `#`, `+`, `.`, `/`, `-`, 공백) 전부를 표에 정의한다. 입력 검증과 폭 계산이 같은 집합을 보므로 "허용되는데 폭을 모르는 문자"가 없다. 오차는 로컬 목업 렌더로 확인해 보정한다.

### 산출물과 README 동기화 (태그 배열이 단일 소스)

- `tag-<slug>.svg`를 저장소 루트(org-*.svg와 동일 위치)에 생성해 커밋한다. `tag-` 접두사는 이 생성기의 예약 이름 공간이다.
- **README의 태그 `<img>` 목록도 스크립트가 갱신한다.** README.md의 마커 구간 `<!-- tech-tags:start -->` ~ `<!-- tech-tags:end -->` 안쪽을 배열 순서대로 재생성한다. 배열/산출물/README 세 곳을 손으로 맞추면 어긋나기 쉽기 때문이다 (생성은 됐는데 README에 안 실리거나, 지운 태그를 계속 참조하는 류).
- **치환 전에 마커가 정확히 한 쌍인지 검증한다** (시작 1개, 끝 1개, 시작이 끝보다 앞). 마커가 없거나 중복이거나 순서가 뒤집혔으면 README도 산출물도 건드리지 않고 오류로 멈춘다. 마커 훼손 상태에서 범위 치환을 강행하면 무관한 README 구간을 지울 수 있기 때문이다.
- 같은 이유로 이번 실행의 배열에 없는 기존 `tag-*.svg`는 삭제한다 (`icons/` 캐시는 다시 쓸 수 있으므로 남긴다).
- 결과적으로 태그 변경 작업 = "배열 수정 + 스크립트 실행"으로 끝난다. README 수동 편집은 없다.

## README 배치

- 위치: 소개 문단 바로 아래, metrics 2단 레이아웃 위에 마커 구간을 둔다.
- 섹션 제목 없이 태그만, 왼쪽 정렬(소개 글과 정렬 통일).
- `align` 속성 없는 순수 인라인 `<img>` 나열: GitHub 패딩 주입이 없고, 좁은 화면에서 자연 줄바꿈된다. 아래 플로트 레이아웃과도 간섭이 없다.
- `alt`는 표시 이름(예: `alt="C#"`)으로 넣는다 (스크립트가 마커 구간을 생성할 때 함께 넣는다).
- 주석 처리된 옛 skillicons 블록(주요 기술/학습 중)은 삭제한다.

## 기각한 대안

- **한 장짜리 스트립 SVG**: 좁은 화면에서 줄바꿈이 안 돼 축소되거나 잘린다.
- **shields.io 배지**: 외부 의존, 스타디움 필 모양이 안 나옴, 팔레트 제약.
- **브랜드 컬러 아이콘**: Unity(검정) 등이 다크 페이지에서 안 보인다.
- **base64 래스터 임베드**: 벡터 소스를 굳이 래스터로 낮출 이유가 없고, 재색칠도 불가능해진다.
- **CI 자동 생성**: 동적 데이터가 없어 트리거가 무의미하고 자동 커밋 소스만 늘어난다.
